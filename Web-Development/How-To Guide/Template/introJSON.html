Working with JSON

See more JSON documentation here

JavaScript Object Notation (JSON) is a syntax for storing and exchanging data between a browser (client) and a server. If you are familiar with Javascript or programming language that contains objects, you are aware that an object has both properties and corresponding values. For example, you might have an object called an 'automobile' that has properties of type make, model, and year, each with their own values. You are likely also aware that you can access these values by using the syntax of your code. Below is a simple example of our automobile object using Java:

JSON works by taking an object and its contents and converts it to a simple-to-use format that allows us to send and receive objects from a server without error-prone and complicated parsing and translation. It also has the added benefit of being able to be used by any programming language. The NASA APIs use JSON to send data in object notation from its server to your application. Later on you will learn how to code your application to retrieve the data from the object that it receives.

Below you will find another simple example using our automobile object again, except this time we will change its notation to be a JSON object and "unpack" it so that we can access its properties. The notation in this example changes slightly. While you will not generally be creating a JSON object yourself as you will be retrieving the JSON objects from a server, you can see that the object is entirely encased in single quotes (denoting it as a single object). You can also see that each property and value is enclosed using double quotes. While Java can be very forgiving with syntax in general, this syntax is required for JSON to work properly.

As it turns out, JSON is a fairly straightforward tool that is extremely useful for data transfer between the client and a server.
